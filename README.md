### Форма Бэкуса-Наура

```ebnf
<program>      ::= { <function> | <statement> }

<function>     ::= [ "interrupt" ] "function" <ident> "(" [ <params> ] ")" <block>
<params>       ::= <ident> { "," <ident> }
<block>        ::= "{" { <statement> } "}"

<statement>    ::= <var-decl> | <array-decl> | <assign> | <array-assign>
                 | <while> | <if> | <print> | <print-num> | <print-str>
                 | <return> | <call>
<var-decl>     ::= "var" <ident> "=" <expr>
<array-decl>   ::= "var" <ident> "[" <number> "]"
<assign>       ::= <ident> "=" <expr>
<array-assign> ::= <ident> "[" <expr> "]" "=" <expr>
<while>        ::= "while" "(" <expr> ")" <block>
<if>           ::= "if" "(" <expr> ")" <block> [ "else" <block> ]
<print>        ::= "print" "(" <expr> ")"
<print-num>    ::= "print_num" "(" <expr> ")"
<print-str>    ::= "print_str" "(" <string> ")"
<return>       ::= "return" [ <expr> ]
<call>         ::= <ident> "(" [ <args> ] ")"

<expr>         ::= <add> { ("==" | "!=" | "<" | "<=" | ">" | ">=") <add> }
<add>          ::= <term> { ("+" | "-") <term> }
<term>         ::= <factor> { ("*" | "/" | "%") <factor> }
<factor>       ::= <number> | "(" <expr> ")"
                 | <ident> | <ident> "(" [ <args> ] ")" | <ident> "[" <expr> "]"

<number>       ::= [0-9]+
<string>       ::= '"' <any>* '"'
<ident>        ::= [a-zA-Z_][a-zA-Z0-9_]*
```

Пример программы (приветствие с чтением имени через прерывания):

```javascript
var done = 0

interrupt function on_key ( ) {
    var c = port_in ( 0 )
    if ( c == 10 ) { done = 1 } else { print ( c ) }
}

print_str ( "What is your name?" )
print ( 10 )
print_str ( "Hello, " )
set_interrupt_handler ( 0 , on_key )
enable_interrupts ( )
while ( done == 0 ) { }
disable_interrupts ( )
print ( 33 )
```

### Семантика

- **Стратегия вычислений**: аппликативный порядок; выражения вычисляются слева направо по стековой модели.
- **Область видимости**: глобальные переменные и локальные переменные функций (все размещаются статически в секции данных).
- **Типизация**: бестиповая — значение есть 32-битное знаковое машинное слово. Тип определяется тем, как значение используется (число, код символа, адрес).
- **Поддерживаемые сущности**:
    - **Число** — 32-битное знаковое целое, диапазон [−2³¹, 2³¹−1]; переполнение приводится по модулю 2³².
    - **Массив** (`var a[N]`) — N последовательных слов в секции данных, индексируется `a[i]`.
    - **Строковый литерал** — хранится в секции данных в формате pstr (первое слово — длина, далее по символу на слово).
    - **Функция** — обычная (с параметрами и возвратом значения) или обработчик прерывания (`interrupt function`, без параметров и возврата).

### Работа с данными

#### Числа

- **Объявление**: `var x = 42` — выделяет слово в секции данных, записывает значение.
- **Арифметика**: `+`, `-`, `*`, `/`, `%` (АЛУ работает только с регистрами).
- **Сравнение**: `==`, `!=`, `<`, `<=`, `>`, `>=` — возвращают `0`/`1` через `CMP` и условный переход.
- **Вывод**: `print_num(x)` — извлекает десятичные цифры через `% 10` / `/ 10`, накапливает в служебном буфере и выводит от старшей к младшей.
- **Вывод символа**: `print(x)` — выводит младший байт значения как символ (`OUT` в порт 1).
- **Ввод**: значение читается из порта прерыванием — в обработчике цифры накапливаются как `acc = acc * 10 + (c - 48)`.

#### Массивы

- **Объявление**: `var a[16]` — выделяет 16 последовательных слов в секции данных (занулены).
- **Запись** `a[i] = v`: транслятор вычисляет адрес `base + i*4` в регистре и выполняет `ST`.
- **Чтение** `a[i]`: вычисляет тот же адрес и выполняет `LD`.

#### Строки (pstr)

- **Объявление/вывод**: `print_str("Hello")` — строка сохраняется в секции данных в формате pstr: первое слово — длина, далее по одному символу на слово. Вывод — цикл по длине с `OUT` в порт 1.
- Перевод строки выводится как `print(10)` (escape-последовательности в литералах не интерпретируются).

#### Встроенные процедуры

| Вызов | Назначение |
|-------|-----------|
| `port_in(N)` | прочитать слово из порта `N` |
| `port_out(N, v)` | записать `v` в порт `N` |
| `set_interrupt_handler(vec, fn)` | назначить `fn` обработчиком вектора `vec` |
| `enable_interrupts()` / `disable_interrupts()` | разрешить / запретить прерывания |
| `memread(addr)` / `memwrite(addr, v)` | прямой доступ к памяти по адресу |

### AST

AST — словари/списки Python, отображаемые в человекочитаемый текст функцией `format_ast` (артефакт `.ast`). Пример для `var x = 2 + 3 * 4`:

```
Program
  body:
    VarDecl x =
      BinOp +
        Num 2
        BinOp *
          Num 3
          Num 4
```

Приоритет операций (`*` глубже `+`) отражён структурой дерева. Корректность AST проверяется в [tests/test_ast.py](tests/test_ast.py).

## Организация памяти

- **Архитектура**: фон Неймановская — код и данные в едином адресном пространстве.
- **Машинное слово**: 32 бита (инструкции кодируются big-endian, слова данных — little-endian).
- **Адресация**: байтовая; память однопортовая (за такт не более одного обращения).
- **Размер памяти**: 32 КБ.

### Разделы памяти

```
       Memory (32 КБ, байтовая адресация)
+----------------------------------+
| 0x0000 : .text                   |  Инструкции программы (до 8 КБ)
|   ...                            |
+----------------------------------+
| 0x2000 : .data                   |  Переменные, локалки, массивы
|   ...                            |  (bump-аллокатор)
+----------------------------------+
| 0x22F0 : INTR_VECTOR_TABLE       |  Таблица векторов (4 × 4 байта)
| 0x2300 : STR_BUF                 |  Служебный буфер цифр для print_num
+----------------------------------+
| 0x2400 : .strings (pstr)         |  Статические строки
|   ...                            |
+----------------------------------+
| 0x4000 : stack (↑)               |  Стек, растёт вверх; указатель R7
|   ...                            |
| 0x8000                           |  Граница памяти
+----------------------------------+
```

### Регистры

```
       Registers
+----------------------------------+
| R0                               |  Обычно держит 0 (база для абсолютной адресации)
| R1 .. R6                         |  Регистры общего назначения
| R7 (SP)                          |  Указатель стека
+----------------------------------+
| Флаги: ZF (ноль), NF (знак)      |
+----------------------------------+
```

### Правила размещения

- Числовой литерал кодируется как immediate-поле инструкции `LI` и в секцию данных не попадает (при значении до 16 бит знаковых; большие константы собираются арифметикой).
- Строковый литерал сохраняется в секции строк посимвольно в формате pstr (длина + символы), в порядке появления.
- Переменные и массивы получают постоянный адрес в секции данных; на регистры не отображаются.
- Стек используется только при вычислении выражений и вызовах функций (через `R7`).

| Категория | Что хранится | Где |
|-----------|--------------|-----|
| число, символ | 32-битное значение | `.data` |
| массив | N последовательных слов | `.data` |
| строковый литерал | длина + коды символов (pstr) | секция строк |
| числовой литерал | в теле инструкции `LI` | `.text` |
| временные значения | результаты вычислений | стек (`R7`) |

### Отображение сложных выражений на регистры и память

Выражения вычисляются по **стековой модели**: операнды кладутся на программный стек (`R7`), бинарная операция снимает два верхних значения в `R1`/`R2`, выполняет АЛУ-операцию и кладёт результат обратно (`R3` — временный регистр в `push`/`pop`). Переменные читаются `LD` и пишутся `ST`.

**Пример**: `var x = 2 + 3 * 4` (фрагмент листинга; видно, что `3*4` вычисляется раньше `2+…`):

```
li r1, #2          ; push 2
st r1, [r7+0x0]
li r3, #4
add r7, r3
li r1, #3          ; push 3
st r1, [r7+0x0]
...
li r1, #4          ; push 4
...
ld r2, [r7+...]    ; pop 4 -> R2
ld r1, [r7+...]    ; pop 3 -> R1
mul r1, r2         ; R1 = 3*4
st r1, [r7+0x0]    ; push результат
```

**Доступ к элементу массива** `a[i]` — адрес `base + i*4` вычисляется в регистре, далее `LD`/`ST` со смещением 0:

```
ld  r1, [r7+...]   ; индекс i
li  r2, #4
mul r1, r2         ; i*4
li  r2, #8192      ; base массива (.data)
add r1, r2         ; адрес элемента
ld  r1, [r1+0x0]   ; чтение a[i]
```

## Система команд

- **Архитектура**: RISC.
- **Длина инструкции**: фиксированная, 4 байта (одно машинное слово).
- **Ввод/вывод**: port-mapped — инструкции `IN`/`OUT` с номером порта (порт 0 — ввод, порт 1 — вывод).
- **Управление**: hardwired.

### Кодирование инструкций

```
 31     24 23   20 19   16 15                0
+---------+-------+-------+-------------------+
| opcode  |  rd   |  rs   |      imm (16)     |
|  8 бит  | 4 бит | 4 бит |      16 бит       |
+---------+-------+-------+-------------------+
```

- **opcode** — код операции (8 бит);
- **rd**, **rs** — регистры приёмника и источника (по 4 бита);
- **imm** — знаковое 16-битное значение: непосредственный операнд, адрес перехода или номер порта.

Кодирование регистров: поля `rd`/`rs` принимают значения 0–7, что соответствует `R0`–`R7`.

### Набор команд

Каждая инструкция выполняется за **2 такта**: `FETCH` (выборка + декодирование) и `EXEC` (исполнение). Вход в прерывание добавляет один такт `TRAP`.

| Код | Мнемоника | Семантика |
|-----|-----------|-----------|
| 1 | `add rd, rs` | `rd ← rd + rs` |
| 2 | `sub rd, rs` | `rd ← rd − rs` |
| 3 | `mul rd, rs` | `rd ← rd × rs` |
| 4 | `div rd, rs` | `rd ← rd / rs` |
| 5 | `mod rd, rs` | `rd ← rd % rs` |
| 6 | `cmp rd, rs` | флаги по `rd − rs` |
| 7 | `ld rd, [rs+imm]` | `rd ← mem[rs+imm]` |
| 8 | `st rd, [rs+imm]` | `mem[rs+imm] ← rd` |
| 9 | `li rd, #imm` | `rd ← imm` |
| 10 | `jmp addr` | безусловный переход |
| 11 | `jz addr` | переход если `ZF` |
| 12 | `jnz addr` | переход если не `ZF` |
| 13 | `jl addr` | переход если `NF` |
| 14 | `jle addr` | переход если `NF ∨ ZF` |
| 15 | `jg addr` | переход если `¬NF ∧ ¬ZF` |
| 16 | `jge addr` | переход если `¬NF` |
| 17 | `in rd, port#imm` | `rd ← port[imm]` |
| 18 | `out rs, port#imm` | `port[imm] ← rs` |
| 19 | `halt` | останов |
| 20 | `iret` | возврат из прерывания |
| 21 | `ei` | разрешить прерывания |
| 22 | `di` | запретить прерывания |
| 23 | `jr rs` | переход по адресу из регистра |

### Режимы адресации

| Режим | Описание |
|-------|----------|
| Регистровая | операнды АЛУ берутся из регистров |
| Непосредственная | значение в поле `imm` (`LI`, переходы, номер порта) |
| Базовая со смещением | `mem[rs + imm]` для `LD`/`ST` (`R0`=0 даёт абсолютную адресацию) |
| Косвенная регистровая | переход по адресу из регистра (`JR`) |
| Портовая | `IN`/`OUT` по номеру порта |

## Прерывания (trap)

Ввод-вывод по варианту реализован через систему прерываний:

- Запрос прерывания проверяется **только на границе инструкций** (в фазе `FETCH`).
- Адрес обработчика хранится в таблице векторов (`0x22F0`); обработчик задаётся на языке через `set_interrupt_handler(vec, fn)` и завершается `iret`.
- При входе сохраняется полный контекст: `PC`, **все регистры и флаги**; при `iret` он восстанавливается.
- Вложенные прерывания **маскируются**: на входе прерывания запрещаются, повторный запрос откладывается и срабатывает после `iret`.
- Данные приходят в порт по расписанию (`[(такт, символ), …]`); «получение данных» и «вызов прерывания» — разные события, буферизация ведётся в модели.
- В каждой строке журнала отображается состояние процессора (`RUN` / `INT` / `STOP`), поэтому видно, идёт ли работа в прерывании.

## Транслятор

Транслятор реализован в [translator.py](translator.py).

Использование:

```
python translator.py <input.alg> <output.bin>

выходные данные:
  output.bin   - бинарный машинный код (образ памяти)
  output.lst   - отладочный листинг <addr> - <HEXCODE> - <mnemonic>
  output.ast   - AST в человекочитаемом виде
```

### Этапы трансляции

1. **Лексический анализ** (`tokenize`) — поток токенов.
2. **Синтаксический анализ** (`parse`) — построение AST.
3. **Генерация кода** (`CodeGen.generate`) — список инструкций + секция данных.
4. **Линковка** (`link`) — разрешение меток в адреса.
5. **Кодирование** (`encode`) — запись бинарного образа, листинга и AST.

## Модель процессора

Модель процессора реализована в [machine.py](machine.py).

Использование:

```
python machine.py <образ.bin> [файл_ввода] [--log <файл>] [--quiet]

  образ.bin   - бинарный машинный код
  файл_ввода  - расписание прерываний: строки "<такт> <символ>"
```

### DataPath

DataPath включает: регистровый файл `R0`–`R7`, АЛУ с флагами `ZF`/`NF`, счётчик команд `PC`, регистр инструкции `IR`, единую память и контроллер портов ввода-вывода. АЛУ-операции выполняются только над регистрами; обмен с памятью — через `LD`/`ST`, с портами — через `IN`/`OUT`.

### Control Unit

Управление **hardwired** — реализовано конечным автоматом в функции `run`. Каждая инструкция проходит фазы `FETCH` → `EXEC` (по такту); вход в прерывание — отдельный такт `TRAP`. Моделирование потактовое и может быть приостановлено на любом такте.

### Регистры и флаги

| Регистр | Описание |
|---------|----------|
| `PC` | счётчик команд |
| `IR` | регистр текущей инструкции |
| `R0`–`R6` | регистры общего назначения (`R0` обычно держит 0) |
| `R7` | указатель стека (SP) |

| Флаг | Условие |
|------|---------|
| `ZF` | результат равен 0 |
| `NF` | результат отрицателен |
